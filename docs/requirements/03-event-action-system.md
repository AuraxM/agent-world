# 事件与行动系统需求设计

> 范围：事件系统（A）、行动系统（B）、决策（C）、多角色互动（D）、玩家干预（E）。
> 立场：纯产品/体验视角，不涉及任何技术方案。

---

## A. 事件系统

### A1. 事件分类（7 个大类）

1. **时间推进类**
   - 例：天亮、午饭时间、宵禁、季节更替。
   - 1 小时步长下，每整点都可能产生此类事件（如 12:00 触发"午餐时间"）。

2. **环境/世界类**
   - 例：下雨、地震、节日游行、市场开张。

3. **社交类**
   - 例：被搭话、被邀请、被赞美、被冷落、关系升降级。

4. **突发类**
   - 例：袭击、意外、撞见、目击。

5. **任务/目标类**
   - 例：被委托送信、家人临终遗言、玩家下发任务。

6. **内部/心理类**
   - 例：肚子饿、孤独、害怕、心血来潮、回忆袭来。

7. **元事件/系统类**
   - 例：NPC 出生、死亡、入场、离场、获得新能力。

### A2. 事件来源

| 来源 | 说明 | 动机/价值 |
|---|---|---|
| 系统周期性 | 时钟推进、节日日历 | 保证世界永不冷场 |
| 玩家手动注入 | 玩家点按某 NPC 让其"被陨石砸中" | **这是玩家唯一的主动交互方式**，必须做到位 |
| 角色行动结果 | A 打了 B → 产生"被打"事件 | 让行动具备"涟漪效应" |
| 多角色互动衍生 | 两人相遇自动产生"相遇"事件 | 让社交自然涌现 |
| 状态变化触发 | 饥饿值越线 → "肚子饿"事件 | 让内部驱动可被统一处理 |
| 环境/位置触发 | 进入墓地 → "毛骨悚然"事件 | 让地图本身有性格 |

### A3. 事件作用范围（scope）

- **单角色私有**：只有当事人知道（如内心活动、私信）。
- **节点内可见**：在同一房间/同一教室的所有人感知。
- **父层级可见**：教室爆炸 → 整个学校都听见（强度 ≥ 4）。
- **子层级可见**：小镇宣布宵禁 → 镇内所有子节点的 NPC 收到。
- **跨层级/全局**：神迹、世界毁灭、节日。
- **关系过滤**：只有"亲密度 ≥ X"的人才会被通知（如家人去世）。
- **能力过滤**：只有"侦查"能力者注意到陌生人鬼鬼祟祟。

### A4. 事件属性需求

每个事件至少应描述：
1. **事件标识与类别**——用于决策时判断重要性。
2. **自然语言描述**——给 LLM 看的"它发生了什么"。
3. **参与者**——主角、对象、旁观者各是谁。
4. **来源**——系统？玩家？某 NPC 的行动？
5. **强度/重要度**（1–5）——决定优先级与记忆持久度。
6. **可见性规则**——见 A3。
7. **时长**——瞬时？持续若干 tick？
8. **是否可被干预/打断**。
9. **建议响应**——可选，给出几个推荐行动。
10. **后续钩子**——结束后会不会自动衍生新事件。

### A5. 事件生命周期

```
产生 → 分发(按 scope) → 被感知 → 进入决策队列 → 触发行动 → 持续(可选) → 结束/消解 → 写入经历(可选)
```

### A6. 多事件并发

**采用合并感知**：当一个 NPC 同帧收到多个事件，一并交给决策，NPC 自己取舍。
但当突发类强度 ≥ 4 时强制中断当前行动。

## B. 行动系统

### B1. 行动来源

每种来源都让 NPC 的"可做之事列表"动态变化：

1. **位置提供**：在饭馆有"点餐/吃饭/结账"，在教室有"上课/举手"。
2. **状态提供**：饥饿时多"找吃的"，受伤时多"包扎"。
3. **能力提供**：会医术 → "救治"；会偷窃 → "扒窃"。
4. **关系提供**：朋友在场 → "请喝酒"；仇人在场 → "挑衅"。
5. **事件提供**：袭击事件附带"逃/围观/制止"。
6. **基础行动**（始终可用）：移动、等待、说话、观察。

### B2. 行动结果可能性

- 产生新事件（被打 → 旁人看到的"打架"事件）。
- 改变自身/他人状态。
- 写入经历（重要程度由行动决定）。
- 改变关系（送礼 → 好感+；背叛 → 信任-）。
- 提升能力。
- 移动位置。
- 创造/消耗资源。
- **失败的可能**：能力为主 + 小幅随机 + 对抗时双方能力比较。
- **成功也分等级**：大成功/普通/勉强，影响后续事件描述措辞。

### B3. 行动时长与并发

- **瞬时 vs 持续**：吃饭可能持续 1 时间步（1 小时），建造房子持续 30 步。
- **可中断性**：吃饭可被"火警"中断；冥想不能被嘲讽中断（除非性格易怒）。
- **并发**：**主动作单线 + 自动反应（如听见、看见）始终在线**。

## C. 决策（LLM Agent）需求

### C1. 决策输入应包括

1. **角色自我画像**：性格 8 维数值 + 描述、当前状态集合、能力清单、关系网（重要他人）。
2. **当前位置环境**：节点描述、父节点、可见的子节点。
3. **触发本次决策的事件**：包括强度与参与者。
4. **可选行动清单**：来自 B1，附来源标签。
5. **同节点其他角色**：他们是谁、与我什么关系、他们在做什么。
6. **近期经历摘要**：而非全部经历，避免噪音；可由"重要度 + 新近度"挑选。
7. **未完成的目标/任务**：长期驱动力。

### C2. 决策应输出

- **选择的行动**（必须）。
- **行动参数**：对谁、用什么工具、说什么话。
- **理由/独白**：用于写入经历，也是玩家观察 NPC 内心的窗口。
- **情绪标签**（可选）：本次决策时的情绪，回写到状态。
- **对事件的"重要度评分"**：自评要不要长期记住。

### C3. 不确定性 / 创造性

**采用方案三：行动类型受限 + 自由内容生成**。
- 行动类型受限（保证系统能处理）
- 行动内容自由（保留 LLM 创造力）

**性格驱动的差异化**是核心体验，必须可靠地、稳定地影响决策结果：
- 同一袭击事件：勇敢者制止、懦弱者逃跑、好奇者围观、共情者救人。
- 同一邀约：外向者欣然同意、内向者找借口推脱。

### C4. 决策异常处理

- **非法选项**：自动回退到"等待/观察"，并记录一次"NPC 发呆"。
- **超时/不响应**：触发兜底行动（默认行为模式：吃饭、回家、闲逛）。
- **自相矛盾**：以行动字段为准，丢弃理由；或重试一次。
- 玩家无论如何不应看到"NPC 卡住"或报错。

## D. 多角色互动

### D1. 决策顺序

**采用并行决策 + 冲突调解**：
- 同一回合所有 NPC 基于同一快照并行决策
- 引擎收集 actions 后检测冲突（同物品、同座位）
- 冲突触发 ConflictEvent 注入下一 tick，让涉及方各自决策

### D2. NPC 行动如何变成他人事件

每个公开行动应自动生成一个对应的"被感知事件"：
- A 打 B：对 A 是"我打了 B"，对 B 是"我被 A 打了"，对围观者是"看到 A 打 B"。
- 同一行动从不同角度产生**不同描述、不同强度**的事件。

### D3. 对话需求

**采用混合方案**：
- 先生成意图，逐轮展开，可被打断
- 平衡拟真度与成本
- 退出条件：达成共识、被打断、一方走开、超过 N 轮

## E. 玩家干预

### E1. 玩家注入事件

**这是玩家唯一的主动交互方式，必须做到位**。

注入维度：选地点、选当事人（可选）、选事件类型、选强度、选可见性、自定义描述。

**多触发入口**：
- 全局浮动按钮 FAB（"⚡ 投放事件"）
- 地图节点右键/长按
- 角色右键/长按
- 快捷键 `E`

**模板库**：内置 ≥ 15 个常用事件模板（环境/突发/社交/任务/氛围）。

**即时反馈**：
- 投放后立刻进入下一 tick 的事件总线
- UI 高亮"⚡ 你投放了：xxx"
- 强度 ≥ 4 触发故事横幅，强度 = 5 触发特写

### E2. 玩家不会做的事

明确不支持以下功能（已确认）：
- 不扮演 NPC
- 不与 NPC 直接对话
- 不附身控制 NPC
- 不替代 NPC 决策

## F. 用户故事示例

### 故事 1：饭馆里的午餐
> 时钟推进至 12:00，系统产生【午餐时间】事件，作用范围=全镇。饭馆内 NPC 张三（INTP，孤僻）感知到该事件，触发"找吃的"内部事件。可选行动列表包含：位置提供的"点餐"、状态提供的"忍一忍"、关系提供的"邀请桌对面的李四一起"。LLM 基于张三性格选择"点餐"+"独自就餐"，理由："我不喜欢和人吃饭"。该理由写入经历。隔壁桌李四（ESFJ）同时感知到事件，看见张三独自吃饭，触发新事件【看到张三独自吃饭】，李四决策：上前搭话。张三收到【李四搭话】事件，被迫面对社交……

### 故事 2：街头袭击
> 玩家手动注入事件：街上 NPC 王五被劫匪袭击。强度 5，scope=节点内可见。三位旁观者：医生（有"救治"能力）、孩童（懦弱+好奇）、武人（勇敢+好斗）。三人并行决策但按反应速度串行落地：武人先冲上去制止；医生看到武人动手后，决定准备救治；孩童看到混战，吓哭跑开。三种性格在同一事件下产生三种戏剧反应——这是核心体验。

### 故事 3：暴雨与心情
> 系统产生【下雨】环境事件，持续 4 小时。NPC 阿琳（INFP，多愁善感）感知后，因性格放大事件强度，触发内部事件【伤感】，写入状态。她原本的行动"去市集"被改为"在窗边发呆"。LLM 输出独白："雨让我想起母亲。"该独白写入经历。两小时后另一 NPC 路过看到她，触发社交事件……

### 故事 4：肚子饿的连锁
> 农民老赵的饥饿值越线，系统产生【内部:饥饿】私有事件。可选行动：回家做饭、去饭馆、偷邻居的菜。老赵性格 = ISTP+贫穷+与邻居关系差，LLM 选择"偷菜"。行动失败（被邻居"警觉"能力发现），衍生新事件【偷窃被发现】，scope=节点内。邻居感知后愤怒，关系恶化。老赵的"偷"行动写入经历，性格中"狡猾"维度微调。

### 故事 5：玩家投放事件
> 玩家在地图上选中"教室3-2"节点，右键 →"投放事件"，选择模板"突然停电"，强度 3，可见性=节点内。下一 tick，教室内 4 个 NPC 都收到该事件。胆小者尖叫并躲到桌下；外向者大声喊"别慌"；好奇者去摸黑找电源；内向者保持安静等待。玩家在事件流看到这 4 条迥异的反应，感受到 NPC 的"活感"。

## G. 开放问题清单

1. 经历容量上限多少？满了如何取舍？
2. 行动失败的反馈：玩家能否看到失败原因？
3. NPC 是否能撒谎？
4. 集体行动：群体逃难、合唱、起哄，是每人各自决策还是有"群体涌现"机制？
5. 历史事件的回响：很久以前的事是否会以"回忆"形式再次成为事件？
6. 决策频率：是每个事件都决策一次，还是定期"打个盹儿"统一处理？
7. NPC 之间是否有"私聊频道"——不让玩家看见的对话？
