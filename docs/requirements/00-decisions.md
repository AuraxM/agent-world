# 关键决策表

本文档汇总产品需求阶段已经收口的全部决策。所有子系统需求文档（01–05）应在本表的约束下被解读。

---

## 玩家定位

- **角色**：纯导演 / 观察者。**不扮演 NPC、不与 NPC 直接对话、不附身控制 NPC。**
- **唯一主动交互方式**：在某个地点指定事件发生（事件注入）。
- **事件注入是 P0 核心玩法**，必须做到位（多入口 + 模板库 + 即时反馈）。

## 时间机制

- **1 步 = 1 游戏小时**，1 游戏日 = 24 步。
- 推进方式：玩家点击"推进 1 小时 / 1 天 / N 天"或自动播放。
- 支持暂停 / 单步 / 倍速 / 快进。

## 地图

- 严格树形结构，最多 4–6 层（世界 > 区域 > 建筑 > 房间 > 局部）。
- 用显式"特殊通道"补充非父子连接（电梯、传送门、密道）。
- **移动规则**：同父节点下兄弟节点视为一步可达；跨父节点必须先回到祖先。
- 节点属性：名称、描述、类型标签、容量、可见性、可访问性、私密度、当前氛围。
- NPC 感知：同节点全见；子节点见入口；父节点听到响动（强度 ≥ 4 的事件冒泡 1 层）。

## 角色

- **性格**：8 维（外向 / 理性 / 进取 / 利他 / 好奇 / 攻击性 / 诚实 / 情绪稳定），值域 -100~+100，UI 同时呈现档位标签。
- **状态**：30+ 项分四类（生理 / 心理 / 社交 / 特殊），三档强度（轻微 / 中度 / 严重），可叠加。
- **经历**：双层——短期 50 条 FIFO + 长期 100 条按重要度晋升。
- **能力**：12 项，5 档（未入门 / 新手 / 熟练 / 精通 / 大师）。
- **关系**：多维（好感 / 信任 / 熟悉 / 依赖），**单向独立存储**，A→B 与 B→A 互不影响。

## 事件

- 7 大类：时间推进 / 环境 / 社交 / 突发 / 任务 / 内部心理 / 元事件。
- 6 类 scope：单角色私有 / 节点内 / 父层级 / 子层级 / 全局 / 关系或能力过滤。
- 强度 1–5，决定可见性扩散与是否中断当前行动。

## 行动

- **类型封闭枚举（约 30 种）+ 内容自由生成**。
- 来源 = 位置 ∪ 状态 ∪ 能力 ∪ 关系 ∪ 事件 ∪ 基础（移动/等待/观察/说话）。
- 行动有时长（短/中/长），主动作单线 + 自动反应（听见/看见）始终在线。
- 决策顺序：同 tick 全员基于同一快照并行决策 + 冲突调解事件。

## 决策（LLM）

- 输入：角色画像 + 当前位置环境 + 触发事件 + 可选行动 + 同节点其他人 + 近期短期记忆 5–8 条。
- 输出：行动类型 + 参数 + 自由文本 + 内心独白 + 情绪标签 + 自评重要度。
- 失败兜底：退回"等待/观察"，写一条"愣神"经历。

## 多角色互动

- 公开行动自动衍生"被感知事件"，不同视角描述/强度不同。
- 对话采用混合方案：先生成意图，逐轮展开，可被打断。

## MVP 范围

- 玩家定位：观察 + 导演（事件注入是 P0 核心玩法）。
- 世界规模目标：≤ 20 NPC × ≤ 50 节点流畅区。
- NPC 死亡：MVP 仅"退场"（可逆），永久死亡 P1。
- 多人：MVP 单机；只读快照分享 P1。
- 内心独白：默认全开。

## 仍待确认的可选项（非阻塞）

- 永久死亡的引入时机（P1 中是否优先）
- NPC 是否能撒谎（可能影响关系系统的真实性）
- 历史摘要由谁生成（LLM 自动 vs 玩家命名）
- "快进摘要"的细节粒度（要让玩家觉得错过的细节"也曾真实发生"）
