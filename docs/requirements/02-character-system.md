# 角色系统需求设计

> 本文档聚焦角色系统的产品需求，从玩家体验与世界真实感出发，不涉及任何技术实现细节。

---

## 1. 性格（Personality）

### 1.1 性格维度
建议在 MBTI 四轴基础上扩展为 **8 个维度**：

1. **外向 ↔ 内向**：决定主动社交还是回避社交。
2. **理性 ↔ 感性**：决定决策依据是逻辑还是情感。
3. **进取 ↔ 谨慎**：决定面对未知时的冒险倾向。
4. **利他 ↔ 利己**：决定是否倾向于牺牲自身利益。
5. **好奇心高 ↔ 好奇心低**：决定主动探索新地点/新人/新事物的概率。
6. **攻击性高 ↔ 攻击性低**：决定冲突中的反应（对抗/回避/调和）。
7. **诚实 ↔ 善变**：决定是否会说谎、隐瞒、操纵。
8. **情绪稳定 ↔ 情绪敏感**：决定状态变化的剧烈程度。

> *动机：八维比四维能产生更立体的角色行为差异，避免"同 MBTI 角色行为雷同"。*

### 1.2 取值方式
**采用方案 C**：连续值 -100 ~ +100 + 显示档位标签。
*体验：兼顾内部精细度与外部可读性。*

### 1.3 性格变化规则
- 默认稳定，长期不变。
- **重大经历**（如至亲死亡、被背叛、宗教体验、长期被善待）可触发某一维度向某方向**缓慢漂移**（每次微调，不突变）。
- 漂移有"惯性阻尼"，避免性格反复横跳。

### 1.4 性格的影响范围
- **行动选择倾向**：内向角色面对"参加聚会/独处看书"会更倾向后者。
- **状态产生概率**：情绪敏感角色更容易产生"焦虑""抑郁"。
- **关系发展速度**：外向 + 利他 角色与他人建立友谊的速度更快。
- **事件解读偏差**：同一事件，攻击性高角色更容易解读为"挑衅"。

## 2. 状态（Status）

### 2.1 状态枚举建议清单（30+ 项）

**生理类（10 项）**：饥饿、口渴、疲惫、困倦、生病、受伤、寒冷、燥热、酒醉、中毒。

**心理类（12 项）**：无聊、专注、兴奋、愉悦、悲伤、愤怒、恐惧、焦虑、羞愧、骄傲、好奇、麻木。

**社交类（8 项）**：孤独、被关注、被忽视、恋爱中、失恋、被信任、被怀疑、社交疲劳。

**特殊类（4 项）**：灵感涌现、宗教体验、顿悟、心流。

### 2.2 产生条件
- **自然衰减/积累**：长时间未进食 → 饥饿；长时间未社交 → 孤独。
- **事件触发**：被攻击 → 受伤 + 恐惧；被表白 → 羞愧或愉悦。
- **行动结果**：饮酒 → 酒醉；阅读 → 专注或无聊（取决于能力与性格）。

### 2.3 强度等级
**采用三档**：轻微 / 中度 / 严重。

### 2.4 消除规则
- **自然恢复**：疲惫 → 睡眠后消失；愤怒 → 时间冲淡。
- **特定行动**：饥饿 → 进食；孤独 → 与他人交谈。
- **特定场所**：受伤 → 医院；焦虑 → 教堂/自然环境。
- **他人介入**：悲伤 → 被安慰。

### 2.5 状态间关系
- **互斥**：愉悦 ↔ 悲伤、专注 ↔ 无聊。
- **共存**：饥饿 + 疲惫 + 孤独 可同时存在。
- **转化**：长期"孤独" → "抑郁/麻木"；长期"愤怒" → "仇恨"（写入关系）。
- **叠加放大**：饥饿 + 疲惫 → 易怒概率提升。

### 2.6 对行动的影响
- 屏蔽某些行动（严重受伤 → 无法奔跑）。
- 改变行动权重（饥饿严重 → 大概率优先觅食）。
- 解锁特殊行动（醉酒 → 解锁"吐真言""斗殴"）。

## 3. 经历（Experience / Memory）

### 3.1 容量
**采用双层结构**：短期 50 条（FIFO）+ 长期 100 条（重要度晋升）。

### 3.2 写入规则
- **触发写入**：完成一次行动、亲历一个事件、获得强情绪状态时。
- **粒度**：一条记忆 = 一次有意义的体验单元。
- **写入频率上限**：避免短时间内被洗刷。

### 3.3 内容结构
每条记忆应包含：
- 发生时间
- 发生地点
- 涉及角色
- 事件简述
- 当时的主观情绪/状态
- 主观重要性评分（角色自己认为多重要）

### 3.4 淘汰策略
- 短期：FIFO，最旧的进入"晋升评估"
- 晋升条件：LLM 自评 importance ≥ 4，或涉及核心关系，或强度 ≥ 4 事件
- 长期：满 100 条按 importance 升序淘汰

### 3.5 对决策的影响
- 决策时优先调用：与当前情境相关的记忆 + 涉及当前角色的记忆 + 高重要性记忆。
- 经历会被"性格"过滤后影响判断（攻击性高的角色更容易记仇）。

## 4. 能力（Ability）

### 4.1 能力维度（约 12 项）
阅读、写作、运动、烹饪、社交、艺术、音乐、战斗、医术、手工、商业、教学。

### 4.2 等级模型
**5 档**：未入门 / 新手 / 熟练 / 精通 / 大师。
- 比纯数值更易玩家感知。
- 内部可保留经验值用于成长曲线。

### 4.3 成长机制
- **行动训练**：执行相关行动后获得经验，达到阈值升档。
- **经历积累**：重大经历（如真实战斗）一次性提升较多。
- **他人传授**：高等级角色教学，加速低等级角色成长。
- **天赋影响**：性格中"好奇心""进取"等维度影响成长速度。

### 4.4 对行动结果的影响
- **成功率**：能力越高，行动失败率越低。
- **产出质量**：同样是做菜，大师级产出"美味食物"，新手产出"勉强能吃"。
- **经验获得**：高等级角色做低难度任务，获得经验更少（边际递减）。

### 4.5 上限与衰减
- 设上限"大师"档位，避免无限膨胀。
- 长期不使用某能力 → 缓慢衰减（运动、战斗类合理；阅读类不合理）。

## 5. 关系（Relationship）

### 5.1 关系类型
家人、恋人、朋友、熟人、陌生人、同事/同学、上下级、师徒、敌人、仇人。
- 类型可叠加（同事 + 朋友）。

### 5.2 强度维度（多维）
- **好感度**（-100 ~ +100）：是否喜欢对方。
- **信任度**（0 ~ 100）：是否相信对方说的话、做的事。
- **熟悉度**（0 ~ 100）：了解对方的程度。
- **依赖度**（0 ~ 100）：是否在情感/物质上依赖对方。

### 5.3 建立 / 发展 / 恶化 / 修复
- **建立**：首次互动 → 记入"陌生人"，熟悉度 > 阈值升级为"熟人"。
- **发展**：正向互动（赠礼、帮助、共度时光）提升好感与信任。
- **恶化**：负面事件（背叛、争吵、被忽视）下降相应维度。
- **修复**：道歉、补偿、长时间良性互动；重大背叛设"修复阻尼"。

### 5.4 对行动选择的影响
- 优先帮助高好感对象。
- 回避高负好感对象。
- 信任度低 → 不接受其建议、不告知秘密。
- 关系类型解锁特殊行动（恋人 → 表白、约会；敌人 → 报复）。

### 5.5 单向 vs 双向
**采用单向独立存储**。
A 对 B 的好感 与 B 对 A 的好感 各自独立。

## 6. 行动（Action）

### 6.1 行动来源
当前可执行行动 = 以下来源的并集：
- **位置提供**：教室 → 上课、传纸条；厨房 → 做饭。
- **状态提供**：饥饿 → 觅食；醉酒 → 斗殴。
- **能力提供**：医术高 → 治疗；战斗高 → 主动挑战。
- **关系提供**：恋人在场 → 表白、亲吻。
- **事件提供**：突发事件强制弹出特定选项（火灾 → 逃跑/救人）。
- **基础行动**（始终可用）：移动、等待、说话、观察。

### 6.2 行动结果种类（可叠加）
- 触发新事件
- 改变自身或他人状态
- 写入一条经历
- 改变关系维度
- 提升/衰减能力
- 移动到新位置
- 失败 / 部分成功 / 成功 / 大成功（受能力与运气影响）

### 6.3 时长概念
**采用"占用时间槽"**（短/中/长三档）：
- 睡眠 = 长；吃饭 = 中；打招呼 = 短。
- 期间该角色不可被打断（除非有更高优先级事件）。

## 7. 角色生命周期

### 7.1 创建方式
- **玩家手动**：自定义性格、能力、初始关系（适合主角）。
- **模板生成**：预设职业模板（教师、医生、学生）+ 微调。
- **随机生成**：用于背景 NPC，保证世界丰富度。

### 7.2 死亡 / 退场
- MVP：仅"退场"（搬走、出国、失踪），可逆。
- P1：永久死亡。
- 角色死亡后，相关角色获得对应"经历"与"状态"（悲伤、孤独）。

### 7.3 身份感（独一无二）
- 每个角色拥有：姓名、外貌简述、出生地、显著特征（口头禅、习惯动作）。
- 性格 + 能力 + 关系网 共同构成"无法复制的个体"。
- UI 上提供"角色档案"页，让玩家形成情感联结。

---

## 8. 玩家与角色的关系

**已确认**：玩家是纯导演 / 观察者，不扮演任何角色，不与 NPC 直接对话。
玩家与角色的关系仅限于：观察、跟随（事件流过滤）、对该 NPC 投放事件。

---

## 9. 开放问题清单

1. **性格漂移阈值**：多大强度的事件、累计多少次才能触发性格漂移？
2. **状态强度的可视化**：玩家是否需要看到 NPC 的所有状态？还是只能从行为推断？
3. **记忆的"主观重要性"由谁评判**：角色自己（被性格影响）还是世界系统客观打分？
4. **能力衰减是否一刀切**：所有能力都衰减，还是只有体力相关的衰减？
5. **关系的"暧昧地带"**：朋友与恋人之间是否需要"暧昧"过渡态？
6. **行动失败的容忍度**：失败后是否给"重来"机制，还是后果完全承担？
7. **NPC 数量上限**：世界规模多大才能保证"涌现"，又不会让玩家信息过载？
