# Web 前端交互与 UX 需求设计

> 本文档聚焦产品/UX 层面：界面应有哪些视图、玩家如何操作、信息如何组织、体验如何流畅。

---

## 1. 主要视图清单

### 1.1 世界主视图（World Overview）
玩家进入游戏后的"家"，一眼看到当前世界全貌与正在发生的事。

| 类别 | 内容 |
|---|---|
| 核心信息 | 当前世界名称与时间（游戏内时间）、当前地图层级位置、活跃 NPC 数量、最近 5 条事件简报、模拟运行状态 |
| 关键操作 | 播放/暂停时间、跳转到任一地图节点、点击 NPC 进入角色详情、**注入新事件** |
| 次要操作 | 切换主题、保存快照、进入设置、唤起搜索 |

### 1.2 地图详情视图（Map Detail）
沉浸式观察某一具体地点内发生了什么。

| 类别 | 内容 |
|---|---|
| 核心信息 | 当前节点名称与描述、父级路径（面包屑）、子节点列表、节点内 NPC、节点内正在发生的事件 |
| 关键操作 | 进入子节点、返回父节点、点击 NPC 查看、**对节点投放事件** |
| 次要操作 | 编辑节点描述、查看该节点历史日志、收藏/Pin 节点 |

### 1.3 角色详情视图（Character Detail）
完整了解一个 NPC 的"人"。

| 类别 | 内容 |
|---|---|
| 核心信息 | 头像、姓名、当前位置、当前状态、性格画像、最近经历、关系一览、能力标签 |
| 关键操作 | **跟随观察**（事件流过滤）、**对此 NPC 投放事件** |
| 次要操作 | 查看完整内心独白日志、查看完整记忆、编辑设定（编辑器模式） |

> 注：移除了"对话""附身"按钮，因为玩家不扮演 NPC、不直接对话。

### 1.4 事件流 / 时间线视图

| 类别 | 内容 |
|---|---|
| 核心信息 | 时序化事件条（时间戳、地点、参与者、事件摘要）、按角色/地点筛选 |
| 关键操作 | 点击事件跳转到该地点/角色、按关键字搜索、按"重要程度"筛选 |
| 次要操作 | 收藏事件为"故事书签"、导出片段、跳转到回放 |

### 1.5 关系图谱视图

| 类别 | 内容 |
|---|---|
| 核心信息 | NPC 节点、关系连线（颜色/粗细表示亲疏/类型）、关系标签 |
| 关键操作 | 点击节点高亮其所有关系、过滤关系类型、查看一对关系的演变历史 |
| 次要操作 | 时间轴回放关系网演变、聚类显示派系 |

### 1.6 决策日志 / NPC 内心独白视图
让玩家看见 LLM 决策的"思考过程"，这是产品独特卖点。

| 类别 | 内容 |
|---|---|
| 核心信息 | NPC 当前面对的事件、其内心思考（LLM 推理输出）、最终行动、所依据的记忆/性格 |
| 关键操作 | 在多 NPC 间切换视角、折叠/展开思考全文 |
| 次要操作 | 导出独白为故事文本、订阅某 NPC 的独白通知 |

### 1.7 世界设置 / 编辑器视图

| 类别 | 内容 |
|---|---|
| 核心信息 | 地图树编辑器、NPC 列表与档案编辑、世界基础设定 |
| 关键操作 | 新增/删除/嵌套节点、新增 NPC 并填写性格、设置初始关系、保存为模板 |
| 次要操作 | 从模板/示例导入、AI 辅助生成 |

### 1.8 历史回放视图

| 类别 | 内容 |
|---|---|
| 核心信息 | 时间游标、可跳转的关键节点列表、当前回放的画面 |
| 关键操作 | 拖拽时间游标、倍速回放、定位到某事件、从此分支重启模拟 |
| 次要操作 | 导出片段为视频/文字故事、分享链接 |

### 1.9 事件投放视图（Inject Drawer）
玩家与世界交互的核心入口。

| 类别 | 内容 |
|---|---|
| 核心信息 | 投放对象（节点 + 可选参与者）、事件类型、强度、可见性、自由文本描述 |
| 关键操作 | 选择模板、填写参数、立即投放、撤销最近一次投放（同 tick 内） |
| 次要操作 | 保存为自定义模板 |

## 2. 地图的视觉呈现

### 2.1 表达"层级"而非"网格"
- 视觉隐喻：**俯视盒中盒**（外层节点像一个房间，里面摆着子节点的"门"或"小盒子"）。
- 不必强制几何坐标准确——节点位置可以由玩家拖拽或自动布局。

### 2.2 进入子节点的交互
**采用三件套**：
- 双击进入（沉浸感）
- 顶部面包屑（保险）
- 可折叠侧栏树（高级用户）

### 2.3 节点内 NPC 的呈现
- **稀疏（≤6 人）**：站位图，头像放在节点画布上，可拖动，气泡显示当前活动。
- **中等（7–20 人）**：头像横排或网格，加状态徽章。
- **密集（>20 人）**：聚合为"人群"图标，悬停展开列表。

### 2.4 跨节点事件提示
- 当前节点之外发生重要事件时，面包屑/侧栏树上对应节点出现**脉冲徽章**。
- 重要事件可触发顶部"故事条"（一行滚动通告）。

## 3. 角色的视觉呈现

### 3.1 角色卡片（多种密度）
- **极简徽章**（地图上）：头像 + 一个状态点 + 名字。
- **小卡片**（列表/侧栏）：头像 + 名字 + 当前活动一句话 + 一个核心情绪。
- **完整卡片**（详情视图）：增加性格、关系、经历、能力、最近独白摘要。

### 3.2 性格可视化
**采用组合方案**：
- 上方雷达图（8 维）
- 下方关键词标签
- 折叠的角色文字描述

### 3.3 状态可视化
- 状态分类显示：**生理 / 情绪 / 社交 / 目标**。
- 每类一行小图标 + 强度条；超过 3 个并存时折叠为"还有 N 个"。
- 强烈情绪以呼吸感动效突出。

### 3.4 经历列表
- 默认按时间倒序，按"今天 / 本周 / 更早"分组。
- 支持按类别筛选（对话 / 行动 / 内心 / 重大事件）。
- 重要经历用书签图标标记。

### 3.5 关系网可视化
- 详情视图里只显示该角色的"一度关系"（自我中心图）。
- 全局关系图谱才显示整个网络。
- 关系连线颜色：暖色=亲密，冷色=疏远，红=敌对，紫=复杂；连线粗细=强度。

## 4. 事件 / 行动的实时反馈

### 4.1 决策反馈节奏
NPC 行动分三段视觉表达：
1. **触发**：事件出现，相关 NPC 头像边缘出现高亮圈。
2. **思考**：头像旁出现思考气泡。
3. **行动**：思考气泡变为对话/动作气泡，事件流添加新条目。

### 4.2 LLM 延迟处理
- 思考期间该 NPC 头像保持思考动画。
- 同时让其他 NPC 仍可被观察。
- 如思考超时（>10s），头像旁出现"思考较深…"温和提示。

### 4.3 多 NPC 并发避免淹没
- 事件流默认**聚合**：同地点 5 秒内的多条事件合并。
- 玩家可调"信息密度"滑杆。
- 重要程度由 LLM 或规则打分。

### 4.4 关注 / 视角跟随
- 任何 NPC 卡片上都有"跟随"按钮。
- 跟随后，事件流自动过滤为"该 NPC 看到/参与/听说的事件"。
- 顶部出现"正在跟随：XXX（× 退出）"标识。

## 5. 玩家操作的核心入口

### 5.1 创建世界引导流程
1. 欢迎页：3 个选项："使用示例世界""选择模板""完全自定义"。
2. 模板页：卡片式（学校日常、中世纪村庄、太空站、家庭剧、办公室政治…）。
3. 自定义引导：分步问答式。
4. 完成：进入主视图，**自动暂停**，提供"按这里开始"高亮提示。

### 5.2 时间控件
- 位置：屏幕**底部居中**，固定漂浮。
- 控件：⏮ 单步回退 / ▶ 播放 / ⏸ 暂停 / ⏭ 单步前进（推进 1 小时） / 倍速 / 当前时间显示。
- 重要事件触发时倍速自动回到 1×。

### 5.3 事件投放入口
**多入口冗余**：
- 全局浮动按钮（FAB）"⚡ 投放事件"
- 地图节点右键 →"在此地点投放事件"
- 角色右键 →"对此 NPC 投放事件"
- 快捷键 `E`

## 6. 信息密度与导航

### 6.1 主屏幕布局
**默认采用 C 方案**（主区 + 抽屉）：中央主区可切换（地图/关系图/事件流），右侧抽屉栏可呼出 NPC 卡 / 独白 / 历史。

### 6.2 移动端
- **最小可用支持**：地图浏览、事件流、角色查看、暂停/播放、投放事件。
- **不建议在移动端做的**：编辑器、关系图谱（信息过密）、复杂多面板。
- 移动端布局：底部 Tab（世界 / 事件 / 角色 / 我）。

---

## 7. 首次使用体验（Onboarding）

- 首次打开应看到一个**已经在运行的示例世界**，自动暂停在第一个有趣的事件上。
- 屏幕上有 3–5 步浮动提示气泡。
- 提供"创建我的世界"和"先逛逛"两个出口。
- 每个核心视图首次进入时显示一次性提示。

## 8. 沉浸感与叙事感

### 8.1 让 NPC "活着"
- 即使空闲，NPC 也有微动作（呼吸、眨眼）。
- NPC 之间偶尔产生玩家未触发的自发互动。
- NPC 拥有"日常作息"——清晨在家、上午上学、傍晚在咖啡馆。
- 内心独白偶尔以"诗意片段"形式弹出。

### 8.2 重要事件的强调
- 三种强调等级：
  1. **轻微**（强度 1–2）：事件流条目高亮 + 微动效。
  2. **中等**（强度 3）：屏幕角落出现"故事横幅"，2 秒后淡出。
  3. **强烈**（强度 4–5）：弹出特写画面（黑色蒙版 + 居中文字 + 慢放头像），玩家点击或 3 秒后退出，时间自动暂停。

### 8.3 叙事性总结
- 每个游戏内"日终"自动生成"今日纪事"。
- 提供"故事模式"入口：把已发生的一切渲染为可阅读的小说草稿。
- 玩家可订阅某 NPC 的"日记本"。

---

## 9. 可访问性与可读性

### 9.1 大量文本阅读
- 长文本（独白、经历）使用"故事字体"，行高宽松、段落分明、最大宽度限制（约 45–70 字符/行）。
- 提供"阅读模式"。
- 重要术语（NPC 名、地点名）自动加色彩标记。

### 9.2 颜色与对比度
- 提供至少**亮色 / 暗色 / 高对比**三种主题，**暗色为默认**。
- 关键状态颜色不仅靠色相区分，也用图标+文字（色盲友好）。
- 字号至少三档可调。

### 9.3 国际化
- 至少中英双语；UI 文案与 NPC 内容文案分离。
- NPC 内容语言可独立于 UI 语言。

## 10. 异常状态的呈现

| 场景 | UI 体验需求 |
|---|---|
| 加载中（启动） | 启动画面带世界名称与一句叙事文案，避免技术感 |
| LLM 决策超时 | NPC 头像下方显示"陷入沉思"，事件流提示，不打断模拟 |
| LLM 决策失败 | 该次决策标记"未知"，NPC 选择"愣神/原地不动"，玩家可点击"重试" |
| 网络中断 | 顶部静默横幅"连接中断，已自动暂停"，重连后提示"已恢复" |
| 世界为空 | 引导式空状态："你的世界还没有 NPC，从这里开始 →" |
| 节点为空 | "这里现在没人……要不要派一个 NPC 过来？" |
| 事件流为空 | "夜深了，世界很安静，要不要按 ▶ 让故事继续？" |
| 保存失败 | 非阻塞提示+重试，绝不直接弹错误代码 |

## 11. 屏幕场景描述

### 场景 A：玩家正在观察"晨曦小镇 > 学校 > 教室 3-2"
> 屏幕暗色基调，宛如夜灯下的剧本。**顶部居中**是面包屑："晨曦小镇 › 阳光中学 › 教室 3-2"，每一级都是可点击的。**左侧约 22% 宽度**是层级树侧栏，当前节点高亮，兄弟节点旁有脉冲点提示"那里也在发生事"。**中央主区**显示教室的俯视画面（卡通风格底图），4 个 NPC 头像散布在课桌周围，其中一人头顶冒着思考气泡，另一人头顶是对话气泡。**右侧约 28% 宽度**是事件流，最新一条高亮闪烁。**底部居中**漂浮着时间控件：当前游戏时间 2026/05/01 14:00，速度 1×。**右下角**有一个紫色 FAB"⚡ 投放事件"。

### 场景 B：玩家点开了 NPC"小红"的角色详情
> 主区被一个**详情面板**从右侧滑入覆盖，约占 60% 宽，左侧仍能看到地图（半透明）。详情面板顶部是大头像、姓名、年龄、当前位置、一句状态。其下分四个标签页：**性格 / 状态 / 经历 / 关系**。当前在"性格"页：上方一个八维雷达图，下方关键词标签，最下面一段角色背景文字。面板**右侧浮动栏**显示她最近的 3 条内心独白片段。**底部**两个主要按钮："👁 跟随她""⚡ 对她投放事件"，次要按钮"📖 完整经历""✏️ 编辑设定"。

## 12. 开放问题清单

1. **沉浸感 vs 工具感**：玩家是把它当"小说生成器"还是"上帝模拟器"？
2. **时间尺度**：1 小时步长下，事件流的密度需不需要每天概要化？
3. **NPC 决策的可见度**：内心独白要默认全部可见，还是只有重要时刻才弹出？
4. **多人协作**：是否需要支持多个玩家同时观察同一世界？
5. **可分享内容**：玩家会想分享哪些瞬间？
6. **付费与限制**：LLM 调用成本不低，是否需要在 UI 上呈现"剩余思考次数"？
7. **教学的强度**：是否做强引导？
8. **历史无限增长**：UI 是否需要"章节化""归档"机制？
9. **音效与配乐**：是否提供环境音？
10. **离线/可恢复性**：玩家关掉浏览器后世界是否继续？
