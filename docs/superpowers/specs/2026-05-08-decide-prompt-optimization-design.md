# Decide Prompt 约束优化

**日期**: 2026-05-08
**状态**: 设计中

## 问题诊断

角色行为决策不合理，具体表现为三类问题：

| # | 问题 | 根因 |
|---|------|------|
| 1 | 饿了/困了/脏了却不行动 | vitals 引导太软（"你最好"），没有优先级框架 |
| 2 | 反复和同一个人对话 | 无反重复约束，不知道上次和谁说了话 |
| 3 | notebook 约定不履行 | 待办放在 prompt 末尾，角色"遗忘" |

额外发现的结构性问题：

| # | 问题 | 根因 |
|---|------|------|
| 4 | 无决策优先级框架 | 饥饿+好友+约定三者冲突时无优先级指导，LLM 随机选 |
| 5 | 行为规则薄弱 | 仅 2 条规则（memorize、禁止编造），无"必须做的事" |
| 6 | 行动选项扁平化 | eat 和 look_around 看起来同等可选 |
| 7 | continuity 信息不足 | 知道上次 action type 但不知道 target，反重复无法执行 |
| 8 | reasoning 约束弱 | 只要求引用性格，不要求解释"为什么选这个行动" |
| 9 | 世界规则偏叙事 | worldRules() 信息密度低，关键约束淹没在文学描述中 |

## 方案

采用**决策优先级框架 + prompt 重排序**的方式，在 user prompt 中注入结构化的决策指导，同时优化静态/动态内容的排列以最大化 prompt cache 命中率。

## 详细设计

### 1. 决策优先级框架

在情绪状态段之后、经济状态段之前，插入优先级指导段：

```
## 决策优先级（严格遵守从上到下的顺序）

### 1. 生理需求（最高优先）
当你感到明显饥饿、明显疲惫或明显不干净时，必须优先解决：
- 饥饿 -> move 去用餐场所（dining 标签）eat
- 疲惫 -> move 去休息场所（residence 标签或 private）rest/sleep
- 卫生 -> move 去洗浴场所（bathing 标签）bathe
生理需求未解决之前，不要做其他事。

### 2. 履行约定
记事本中如果有当前时段或即将到期的待办事项，优先赴约。
约定好的事不去做，就是不守信用。

### 3. 社交适度
不要连续两 tick 对同一个人 speak。
如果你上一 tick 刚和某人说过话，这 tick 换个人或者做别的事。

### 4. 自由行动
以上都不触发时，根据你的性格、目标、感知到的事件自由选择。
```

**配套删除**：
**阈值说明**：优先级 1 在 vitals 数值 > 8 时触发（饥饿/疲惫/卫生任一 > 8），对应 qualifyVital 的 "明显饥饿/疲惫/不干净"（moderate 档上沿接近 high）。比原来的 urgency 提醒（>= 10，high 档）更早介入。

**配套删除**：
- 移除 "你最好XXX" vitals 引导段（上次改动新增）——已被优先级 1 覆盖
- 移除 ⚠ urgency 提醒段（当前 L1368-1393）——已被优先级 1 覆盖，且优先级 1 阈值更低更早触发

### 2. 行为规则重构

从当前 2 条规则扩展为"必须/建议"两层：

```
## 行为规则

### 必须遵守
- 生理需求（饿了吃、困了睡、脏了洗）是本能，不由性格左右。
  不论你是外向还是内向、勤快还是懒散，该吃饭时必须吃饭，该睡觉时必须睡觉。
- 禁止连续两 tick 对同一个人 speak。换个人说话，或者做别的事。
- 禁止编造不存在的约定、任务、计划或人物。只依据记忆和真实经历做判断。
- 记事本上有待办事项时，必须在当前时间段内规划执行。不可无故拖延。

### 建议遵守
- 与人互动后产生了新印象或了解到重要信息时，调用 memorize 记录下来。
- 不确定对方是谁时，先 recall 查询，不要假装认识陌生人。
```

**配套变化**：companionship 段的"不要假装认识陌生人"移除此处统一管理。

### 3. Notebook 位置提前

- `upcomingNotebookText` 从 prompt 末尾（当前在 arrivalIntro / 全局事件之后）移到决策优先级段正下方，紧跟"约定履行"优先级
- `describeEntries` label 从 `待办：` 改为 `约定的待办：`，强化"这是约定"的暗示

**新 prompt 局部结构**：
```
## 决策优先级
...
### 2. 履行约定
记事本中如果有...

约定的待办：
- 2小时后 — 和王阿姨约好在茶馆喝茶

你的经济状态：
...
```

### 4. 信息支撑

让约束"可执行"：

**4a. `AggregatedFacts.lastAction` 增加 `targetId`**

`describeContinuity` 当上一行动是 speak 时渲染目标名字：
```
- 上一 tick 你的行动：speak，对象：王阿姨："今天天气真好啊"
```

**4b. `todaySpeakCounts`**

在 `AggregatedFacts` 中新增，统计今日每个角色的对话次数。在连续性段展示：
```
- 今日已对话：王阿姨 3 次，李大叔 1 次
```

让角色自己看到"已经和王阿姨说了 3 次了"。

### 5. Prompt 顺序重排

**原则**：静态内容尽量靠前，动态增量尽量靠后，最大化跨 tick 的 prompt cache 命中。

调整后的 user prompt 顺序：

| 序 | 段 | 性质 |
|----|-----|------|
| 1 | 身份锚点 | 静态（同角色跨 tick 不变） |
| 2 | 角色静态认知 | 静态 |
| 3 | 目标 | 静态 |
| 4 | 决策优先级框架 | 静态（新，从后面提前） |
| 5 | 行为规则 | 静态（从后面提前） |
| -- | **缓存断点** | -- |
| 6 | 连续性（含 speak 计数） | 动态 |
| 7 | 当前位置 | 动态 |
| 8 | 生理状态 | 动态 |
| 9 | 情绪状态 | 动态 |
| 10 | 约定的待办 | 动态（从末尾提前） |
| 11 | 经济状态 | 动态 |
| 12 | 同节点人物 | 动态 |
| 13 | 感知事件 | 动态 |
| 14 | 记忆 | 动态 |
| 15 | 可选行动 | 半静态 |
| 16 | 调用规则 | 半静态 |
| 17 | 当前时间 | 动态 |
| 18 | 全局事件 | 动态 |
| 19 | 提交指令 | 静态 |

**关键变化**：
- 决策优先级 + 行为规则 从动态区提到缓存断点之前 → 同一角色跨 tick 命中缓存
- notebook 从末尾提前到经济状态之前 → 紧跟决策优先级，形成呼应
- 移除了 urgency 提醒段和 vitals 引导段（已被决策优先级覆盖）
